Reusable generic execution contract for one governed run_## at a time. This file governs saved run packages: what must be preserved, how a kept run stays reopenable, how closure stays honest, and how saved artifacts stay comparable across reruns. Family choice, winner promotion, and broader benchmark pressure stay upstream.

4.4 Countability and freeze gate what becomes a real governed run and when the next one may start 
countability and freeze
  -> preserve one governed comparison unit
  -> promote only with full bundle
  -> freeze before the next governed run
  -> publish countable kept run = yes / no
  -> do not inflate probes into governed runs
Gate severity: if freeze, checklist, or binary countability publication is skipped, later drift is an execution defect. A run cannot become countable retroactively through nicer prose or a stronger final HTML.
Counting rule MUST read it this way
Governed run meaningA governed run is a preserved comparison unit such as run_01 or run_02, not every raw command or probe inside it.
Intermediate attemptsPromote one into a governed run only if all are true: distinct business or trust delta exists, dedicated validation bundle exists, dedicated SUMMARY.md exists, dedicated score support exists, and later package accounting counts it explicitly inside the active window.
Strong diagnostic, incomplete bundleIf a diagnostic changed causal understanding but still fails the promotion bundle, keep it only in runs/run_##/atf/iteration-ledger.md. Add only the narrowest supporting validation/context note needed to explain winner honesty. If it changes package-level truth or standing, also name it in CALIBRATION.md. Do not count it as a governed run.
Freeze ruleOnce a kept run turns green, freeze its summary, review, validation, and manifest before continuing.
Next-run lockDo not start the next governed run until the current run has a frozen package, an artifact-checklist.md, and a binary countable kept run = yes / no verdict.
Countable meaningUse countable kept run = yes only when the frozen run can be reopened cold as one preserved comparison unit with identity, prompt provenance, review support, and decisive proof.
Not-countable meaningIf the run taught something but stayed under-preserved, publish countable kept run = no, say why, and keep it as history-only, diagnostic-only, or corrected-assumption memory instead of active accounting.
Anti-inflationDo not let repeated probes inside one kept run look like extra governed runs.
