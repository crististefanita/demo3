Legend

2.3 Governed Run Loopthe repeating lane after launch is earned
Legend
  • Scope: this is the only repeating zone.
  • Freeze run_n: produce run-local truth before any `run_n+1` starts.
  • Read run truth: consume saved pretrain and ledger expectations. Do not rebuild a winner from memory.
  • Write next-run truth: usually means `review/next-run-eligibility-card.md` when another governed run is still possible.
  • Legal outputs: same-round continue. Fresh rerank. Blocked. Exit to close-out.
  • Close-out: not another run. It is the terminal state of the current governed window.
2.4 Atomic Next-Run Lockthe binary lock that must close before any later governed slot is legal

  • countable kept run = no: history may stay. Score power may not.
  • Next step: if another governed run is still possible, write review/next-run-eligibility-card.md only after this lock is closed.
2.5 Failure Decoderthe cold decision tree when the lane does not pass cleanly
Legend

  • Read each row as: observed failure -> mandatory check -> immediate action -> legal stop shape.
  • MUST repair local missing artifacts in the same execution when that is still possible.

3.1a Rigid Checklist Lockfreeze must become checklist truth before countability or next-run legality exists

3.2 Run Truth Readingwhat one kept run is allowed to prove and what it is not allowed to fake
Legend
  • RUN_EXPECTED_DELTA_QUALITY: decides whether the kept run reads as verdict-moving.
  • Expected vs actual: read it against saved pretrain or ledger expectations. Do not rebuild it after the run.
  • Novelty role vs materialization: keep them separate. A role label must not smuggle in materialization class.
  • ROUND_TARGET_TEST_COUNT: not a one-run pass/fail. It lives later in package accounting.
  • Source scope + source owner: they are a pair. Support carriers without one authoritative verdict owner cap source authority.
  • If countability = no: the run may still teach something. It must not silently drive accounting, score posture, or HTML strength claims.
  • Important: a kept run may clear run-quality and still stay below promotion after accounting, yield, source, or reporting penalties.
3.3 Artifact Taxonomy And Owner Chainwhat must be visible, what stays memory-only, and what can be deleted
Legend
  • Checklist artifacts: mandatory visibility by contract.
  • Extra artifacts: mandatory only when final HTML really relies on them in a claim, closure reading, or authority cap.
  • Memory-only residue: stays `no`. Existence alone does not upgrade it.
  • When source truth exists at more than one layer: publish `source carrier scope = run-local / root-level / package-level`.
  • Also publish: one `authoritative source verdict owner = scope + artifact path`.
  • Source posture: `strong local` needs local decisive support plus a local owner.
  • `partial`: keeps an explicit cap.
  • `capped`: source truth is still too thin to lift package authority.
  • When an extra artifact carries a published claim: its HTML owner must be exact. Use `canonical section + local anchor`, not a vague family like `5.5`.
  • Checklist examples: `review/artifact-checklist.md`, `review/next-run-eligibility-card.md`, `analysis/canonical-render-check.png`, `analysis/local-link-check.txt` when final publication changed.
  • Claim-relied extra examples: `review/strategic-reopen-notes.md`, `review/pruned-run-memory.md`, `review/source-delta-carrier.md` only when final HTML really leans on them.
  • Memory-only or removable examples: stale `runX` folders with missing decisive proof, duplicate logs with no new reopen value, or side notes with no published claim and no closure reading.
Legend
  • strong local: local support and local owner both exist.
  • partial: support exists, but authority still lives higher.
  • capped: support or owner path is too thin to claim settled source authority.
3.4 Zero-Kept / Falsifier-Only Statewhat the lane is still allowed to do when nothing countable survived yet
Legend
  • countable kept = no: the run does not vanish. Its history and falsifier value still stay auditable.
  • It also does not mean: package close-out, final HTML, or strategic completion. The lane may still continue, rerank, or hard-block after root truth is repaired.
  • Root/package carriers: update them now if the demotion changed package truth.
  • Writes now: the demoted run checklist plus any stale root carrier touched by the demotion. Typical examples: README.md, CALIBRATION.md, review/package-close-out.md, review/frozen-package-inventory.md.
3.5 Root Spine Repair Before Continuationwhen later runs are blocked until root package truth is repaired first
Legend
  • Main law: a good run spine does not license stale package truth.
  • If root carriers became false: repair comes before continuation.
  • Why: this prevents drift where run truth stayed usable while root truth lagged behind.
  • Continuation: judge it again only after the repaired root spine is on disk.
  • Writes now: the exact stale root carriers. Typical examples: manifests/package-manifest.md, review/frozen-package-inventory.md, review/package-close-out.md, and any root claim carrier that had become false.
3.6 Branch-Memory Updatehow the branch-level README keeps active ROOT_RUN truth cold
Legend

  • It must publish: active ROOT_RUN, package state, and the cold next step.

Legend

4.2 Next-Run Lawwhat continuation is allowed to do after one kept run
Legend
  • One low-novelty run: does not auto-continue. Same-round continuation still needs copied-row truth from the ledger.
  • If `winner owned` is true but the remaining frontier set is not reconciled: `copied frontier row = n/a` is honest.
  • Then: rerank becomes mandatory.
  • Two low-novelty runs: make the continuation bar materially harder. The next slot must beat stronger frontier pressure coldly or stop spending.
  • Exit to close-out: can be correct even while application-level exploration remains open. Same-round stop does not mean the whole application is exhausted.
  • Writes now: review/next-run-eligibility-card.md for same-round truth, or a fresh review/frontier-ranking-ledger.md when copied-row truth has already collapsed.
4.2a Same-Round Stop To Fresh-Round Restartblocked continuation does not mean exploration is exhausted
Legend
  • Blocked same-round is a package-stop truth, not an application-exhaustion truth.
  • If opportunity remains, the legal restart is a fresh round with fresh pretrain and fresh rerank.
  • No carry-over winner: the previous winner does not authorize the next fresh round automatically.
4.3 Proof Species And Expected SpineAPI-only, UI-visible, and mixed runs do not owe the same proof bundle
Legend
  • API-only: not proof-light.
  • It still needs: decisive validation, source truth, and one decisive human-readable proof carrier.
  • Page-level render proof: may stay honest n/a only when the run species truly does not rely on canonical page visuals.
  • UI-visible: expect browser-facing proof such as Extent, screenshots, or another named equivalent human-readable proof carrier.
  • Mixed: business truth is partly API and partly UI. One side cannot be treated as decorative support for the other.
  • Publication proof: separate from run proof.
  • Examples: `canonical-render-check.png` and `local-link-check.txt` belong to final-page verification, not to every single run.
  • Cold law: native Extent may be absent.
  • But: decisive human-readable proof may not disappear for counted kept runs.
4.3a Human-Readable Proof MatrixExtentReport yes/no is not the same question as proof carrier yes/no
Legend
  • Two separate questions: was native Extent saved, and does a decisive human-readable proof carrier exist?
  • Named equivalent is allowed: only when it is clearly decisive and human-readable.
  • Missing means pressure: page render proof does not fill the same slot for counted kept runs.
4.4 Sensitive False Readingsfailure rules
Legend
  • What it illustrates: three false readings that sound plausible under pressure. Publication drift. Continuation drift. Accounting drift.
  • Read each row as one fixed grammar: observed state -> false reading -> cold check -> allowed conclusion.
  • MUST keep run truth separate from package close-out truth until close-out owners are actually frozen.
  • MUST recheck copied-row frontier truth before same-round continuation; a proved winner does not automatically donate rank to the next slot.
