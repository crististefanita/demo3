8.4 Run-Quality Gate JUDGE the shared threshold and formula for verdict-moving run quality 
run-quality schema
  -> primary run gate
     RUN_EXPECTED_DELTA_QUALITY

  -> computed run result
     RUN_DELTA_SCORE_ACTUAL

  -> secondary round accounting
     ROUND_TARGET_TEST_COUNT
RUN_DELTA_SCORE_ACTUAL
  = 10 * (
      0.30 * T
    + 0.25 * B
    + 0.20 * P
    + 0.15 * H
    + 0.10 * C
    )

where
  T = truth delta
  B = branch / depth delta
  P = proof strength delta
  H = benchmark / promotion honesty delta
  C = closure trust delta

clear the run-quality bar only when
  RUN_DELTA_SCORE_ACTUAL >= RUN_EXPECTED_DELTA_QUALITY
  and T >= 0.67
  and B >= 0.50
  and P >= 0.75

if T or B fails but P/H/C are strong
  -> countable proof may remain valid
  -> verdict-moving = no
  -> same-branch continuation requires a new frontier target or deeper materialization
Scoring item Role Cold reading
RUN_EXPECTED_DELTA_QUALITYPrimary run gate.The kept run must clear this before it reads as verdict-moving.
RUN_DELTA_SCORE_ACTUALComputed support behind the gate.Do not replace it with softer overall-feel language.
ROUND_TARGET_TEST_COUNTSecondary accounting target.Tracks cold-distinct defended scope materially turned into tests, not raw test volume; one local test does not satisfy it by itself, and it does not replace winner quality.
TTruth deltaDid the run materially change what the round can honestly claim?
BBranch / depth deltaDid the run defend deeper branch scope, not just another sample?
PProof strength deltaDid the run materially improve direct proof precision?
HBenchmark / promotion honesty deltaDid the run improve comparative honesty rather than over-promotion?
CClosure trust deltaDid the run materially improve closure trust, not only polish?
