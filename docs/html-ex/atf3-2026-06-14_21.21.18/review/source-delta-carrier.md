# Root source delta carrier

- Root diff-ul trebuie citit ca pachet de auto-servire autentificata, nu ca suma mecanica a cinci clase.
- Cele cinci run-uri se leaga in aceeasi poveste business:
  - intrare in cont
  - folosire durabila a mesageriei
  - contracte reale de plata
  - absenta istoricului post-purchase
  - owner gate final prin parola + TOTP

- De aceea root diff-ul compileaza:
  - pagini si servicii de cont
  - servicii si validari de mesaje
  - servicii si validari de checkout/plata
  - suportul TOTP
  - clasele de test care materializeaza coridorul
