# Round Pretraining Brief

## Ce surse au contat rece

- runtime-ul local a fixat obligatia `5` teste bune per run si `5` run-uri consumate per pachet
- `docs/out/README.md` a fixat ca ultimul pachet local care conteaza este `2026-06-14_20.03.55`
- ultimul pachet local a fixat interdictia:
  - nu este legal sa revin cu alta variatie pe aceeasi absenta de invoice dupa plata
- `html-ex` a ramas benchmark pentru forma, severitate si audit, nu pentru business-ul meu

## Ce iau si ce nu iau

- iau:
  - disciplina de freeze complet
  - folosirea CSS comun
  - graph business-first
  - diff mic per run + diff mare pe pachet
- nu iau:
  - business-ul benchmark-urilor
  - limbaj comparativ in raportul final
  - supra-notare pentru un pachet privat mai ingust

## Ce familie bate rece acum

- castigator: `incredere privata autentificata si autogestiune securizata`
- motive:
  - rupe dominanta checkout/comert din pachetul precedent
  - foloseste 5 familii locale deja materializate in repo
  - poate atinge targetul activ pe fiecare run fara artificii

## Cum inchid contoarele

- `tests produced`:
  - fiecare run este ales doar daca are `>= 5` teste reale
- `runs consumed`:
  - toate cele `5` sloturi sunt planificate upfront si vor fi inghetate complet
