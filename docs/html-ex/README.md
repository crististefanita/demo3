## Reading Rules

- Do not treat this library as rigid law.
- Prefer the strongest package for the specific family you are calibrating.
- Prefer packages with real positive persistent objects over blocker-only packages when both teach the same lane.
- Prefer blocker packages only when they explain a missing object more inevitably than the positive packages can.
- Read each package cold from its final HTML report first, then its `README.md`, then the strongest run artifacts.
